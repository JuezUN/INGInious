Understanding your result

After you submit your code, it will be assessed and a result will be given to you.
The tables below list all possible results.
Results related to your code. Results you may receive after assessing your code

  • ACCEPTED

    Awesome. Everything worked as it should have: your code compiled/ran and passed all the test cases. Congratulations!

  • WRONG ANSWER

    Sorry, but your code failed to solve the problem. Your code does not have any syntax errors, but it does not solve the problem you were tasked for.
    The output of running your code, what it prints to the screen, is different to the output the system expected.
    If the administrator allowed it, you may be able to see the difference between your output and the expected output.
    Tips: Read carefully what you are asked to do and check again your code.
    Test your code with different inputs. You can do it in your machine or you can use the "custom input" tool from the tools section.

  • COMPILATION ERROR

    Ugh, your code does not compile :/. Something is faulty in your code: it may be a misplaced colon, bracket, quotation mark, or symbol; you may be using a reserved keyword as a variable; you may have chosen a different programming language; or anything, really.
    The "compilation error" message includes the output that the compiler produced with the error. Read the message carefully, it is problably telling you what went wrong.

  • RUNTIME ERROR

    "Runtime error" means that your code failed while it was executing, which means that your code compiled but exited unexpectedly when executed. There are many reasons why your code could crash, some of them are: division by zero, invalid memory access (e.g. bad index access to an array), and null pointers.

  • TIME LIMIT EXCEEDED

    Time limit means that your code exceeded the time limit. This means that your code took too long to run the input test cases. There are two options: either your code got stuck in an infinite loop or your code does solve the problem but takes much longer than it should.
    Tips: Check and debug your code as it may be stuck in a loop. If your code does in fact solve the problem but it is too slow, try to find another solution, one that takes less time to compute.

  • MEMORY LIMIT EXCEEDED

    Getting this result means your code is using too much memory at runtime.
    Tips: Check the data structures you are using, or find a more "optimal" solution, a solution that uses less memory.

  • OUTPUT LIMIT EXCEEDED

    Getting this result means the output of your code is longer that the allowed.
    Tips: Check for printing statements in a loop.
Results related to UNCode. Unexpected errors when your code was been executed.

  • INTERNAL ERROR

    If you get this message, do not worry, There's Nothing Wrong With Your Code!
    This error means UNCode had a problem while running or assessing your code. So, the problem is on our side. We apologise in advance.
    Please reload the page and try submitting again! If you still get the same result, please wait a while and try again or contact your professor.

  • SUBMISSION TIMED OUT

    If you get this, it means that we had a problem running and assessing your code. Wait for a while and try to send a submission again. This might be also caused by a too small time limit in the task configuration (talk to your professor about it).

  • OVERFLOW

    If you get this, it means the grading environment run out of memory. There might be two possible causes: 1. the configured memory is to low to solve this problem (talk to your professor about it). 2. Your code is printing too much data, possibly caused by prints in an infinite loop.

  • ENVIRONMENT NOT
    AVAILABLE

    There was a problem while assessing your submission. Please wait a while and reload the page, and try submitting again.

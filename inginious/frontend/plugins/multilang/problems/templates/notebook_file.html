What is supported in your Notebook?

What is supported in your notebook?

In general, when submitting a notebook, all python 3.5 code is supported on UNCode. However, Ipython notebooks and python code differ in some functionalities, then UNCode might support them or not. Also, there are some things you should bear in mind when creating your notebook.
  • Additional python modules:

    You are able to use the next python modules in addition to the pre installed python modules:
    import pandas, numpy, matplotlib, scipy, scikit-learn, seaborn, datascience, plotly, cufflinks, bokeh, tabulate, nltk

    In case you want to use one module and it is not in the previous list, do not hesitate on reaching out for support.
  • Not internet access from the grading environment:

    When you submit your notebook, take care of not using code that requires internet access. For instance installing a python module or installing data with nltk.download(). All the modules and data you require are already installed and supported by UNCode.
    Then, when you have lines installing a module, for instance: !pip install pandas These lines are automatically removed from your notebook to avoid runtime execution errors.
  • Magic functionality execution:

    Magics function are automatically removed from your notebook. For example the next line:
    %matplotlib inline In case your notebook contains some lines like this, they will be automatically removed to avoid runtime execution problems.
  • Commands execution:

    As notebooks provide a functionality to execute commands like these:
    !ls
    !pwd
    These kind of lines are transformed to be able to run the commands, for example !ls is turned into the following code:
    import subprocess
    call = subprocess.run(["ls"], stdout=subprocess.PIPE)
    print(call.stdout.decode('utf-8')
     That code will do the same for you as in the notebook. However, you are not allowed to assign the result of command execution to a variables, for instance: result = !ls. In that case, your code will return an error.
  • Reading files from Google Drive:

    When you use Google Colaboratory (Colab) and you want to load and read data from Google Drive, you are required to use some python modules unique in Colab. These modules are not supported by UNCode, thus, make sure when you submit your notebook, you do not include it.

    In order to read data, for example a .csv file, you should run your notebook locally using Jupyter.
